# 쇼핑몰 연동

## 프로젝트 개요

> **사용자 친화적인 이커머스 경험 제공을 위한 시스템 연동**

- **목표**: 각기 개발된 상품 목록, 상세 페이지, 장바구니, 결제 모듈을 하나로 통합하고 페이지 간 데이터 끊김 현상을 해결함.
- **주요 기능**:
    - 리액트 라우터를 이용한 SPA(Single Page Application) 구조 설계
    - `useParams`를 활용한 동적 상품 상세 페이지 구현
    - `location.state`를 이용한 결제 데이터(금액) 전송
    - 조건부 로직을 활용한 배송비 자동 계산 (10만원 이상 무료)
**참여 팀원**: 안영진

---
## 개발 환경

- **Language**: JavaScript (ES6+)
- **Framework**: React (v18.x)
- **Library**:

## 프로젝트 구조
기능 단위로 컴포넌트를 분리하여 유지보수성을 높였습니다.

- **App.js**: 라우터 설정 및 전역 상태(addedItems, cards) 관리

- **ProductList.js**: 전체 상품 그리드 렌더링 및 장바구니 담기 토글

- **ProductDetail.js**: 개별 상품 정보 및 동일 브랜드 상품 추천 섹션

- **Cart.js**: 선택 상품 리스트 및 총 결제 금액(배송비 포함) 계산

- **Wallet.js**: 등록 카드 목록 표시 및 최종 결제 승인 처리

- **Success.js**: 결제 결과 피드백 페이지

## 주요 함수와 기능

- **handleAddToCart(id)**: App.js에 위치. 장바구니 배열에 아이템을 추가하거나 제거하며 전역 상태를 동기화함.

- **relatedProducts 로직**: ProductDetail.js 내에서 현재 조회 중인 상품의 brand와 일치하는 상품들을 필터링하여 노출.

- **배송비 계산 로직**: Cart.js에서 reduce 함수를 통해 합계를 구하고, sum >= 100000 ? 0 : 3000 조건문 적용.

- **데이터 클렌징**: 결제 완료 후 setAddedItems([])를 호출하여 장바구니를 초기화함.

## 테스트 방법

기능별 동작 확인을 위해 아래 시나리오를 수행합니다.

- **장바구니 유지 테스트**: 상품을 담은 후 상세 페이지로 이동했다가 다시 돌아와도 '담김' 상태가 유지되는지 확인.

- **무료 배송 테스트**: 합계 금액이 10만원 미만일 때 배송비 3,000원이 합산되는지 확인.

- **결제 흐름 테스트**: 카드 등록 후 해당 카드로 결제 시 Success 페이지에 정확한 총액이 전달되는지 확인.

```bash
npm test
```

## 유의사항 및 알려진 이슈

- **새로고침 초기화**: 현재 브라우저 메모리(state)에 데이터를 저장하므로, 새로고침 시 데이터가 초기화됨. (향후 localStorage 도입 예정)

- **이미지 경로**: /images/ 폴더 내 이미지 파일명이 대소문자를 구분하므로 배포 시 주의 필요.

- **추가 예정 기능**: 상품 검색 기능 및 카테고리 필터링 기능.
