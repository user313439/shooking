# 쇼핑몰 연동 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
SHOOKING MALL 서비스 연동 및 상세 페이지 고도화

### 사용하게 될 기술: 
- React (v18.x)
- JavaScript (ES6+)
- CSS3 (모바일 반응형 레이아웃)

### 문서 작성자: 
안영진

## 📝 고객 요구사항 정리

1. **페이지 간 연동**: 상품 목록, 장바구니, 결제 페이지 간 이동이 끊김 없이 자연스럽게 이루어져야 함.
2. **데이터 일관성 유지**: 페이지를 이동하더라도 사용자가 장바구니에 담은 상품 데이터가 유지되어야 함.
3. **상세 페이지 구현**: 상품 클릭 시 상세 정보와 함께 같은 브랜드의 '관련 상품' 추천 목록이 표시되어야 함.
4. **결제 완료 처리**: 결제 완료 후 총 결제 금액이 포함된 완료 페이지가 노출되어야 함.
5. **테스트 URL**: 실제 구동 환경을 확인할 수 있는 테스트 URL 배포 필수.

## 📋 기능 목록

### 1. 페이지 연동 및 데이터 관리
- **[P0] 전역 상태 관리**: 상품 목록에서 담은 아이템이 장바구니와 상세 페이지에서도 동일하게 유지되도록 구현.
- **[P0] 라우팅 시스템**: 각 페이지(목록 ↔ 상세 ↔ 장바구니 ↔ 결제) 간의 뒤로가기 및 이동 버튼 동작.

### 2. 상품 상세 페이지 (Detailed View)
- **[P1] 상품 정보 노출**: 선택한 상품의 이미지, 브랜드명, 가격, 상세 설명 표시.
- **[P1] 브랜드 관련 상품 추천**: 현재 상품과 동일한 브랜드의 다른 상품들을 하단에 리스트로 노출.
- **[P2] 수량 선택**: 상세 페이지 내에서 구매 수량을 조절할 수 있는 기능.

### 3. 장바구니 및 결제 시스템
- **[P0] 실시간 금액 계산**: 수량 변경에 따른 상품 합계 및 배송비(10만원 기준) 자동 계산.
- **[P1] 카드 결제 모듈 연동**: 기존 개발된 카드 선택 및 등록 창 호출 기능.

### 4. 결제 완료 페이지 (Order Success)
- **[P1] 결제 결과 리포트**: "결제 완료!" 문구와 함께 최종 결제된 총 금액 표시.
- **[P2] 홈으로 이동**: '상품 목록 보기' 버튼을 통해 초기 화면으로 돌아가는 기능.
