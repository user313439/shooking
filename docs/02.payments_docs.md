# 결제 모듈 프로젝트

## 프로젝트 개요

**목표**: 사용자 중심의 직관적인 카드 등록 및 결제 흐름 구현
- **주요 기능**: 
  - 실시간 카드 번호 마스킹 및 포맷팅 (9~16자리 마스킹 처리)
  - 유효성 검사를 통한 '작성 완료' 버튼 활성화 로직
  - 보유 카드 목록 관리 및 새로운 카드 추가 기능 (CRUD 기초)
- **참여 팀원**: 안영진

## 개발 환경

- **Framework**: React 18.x
- **Language**: JavaScript (ES6+), CSS3
- **Deployment**: Vercel

## 설치 및 실행 방법

```bash
npm install
npm start

## 컴포넌트 구조

- **App.js**: 전체 페이지 라우팅(상품 목록/보유 카드/카드 추가) 및 카드 데이터 상태 관리 담당

- **PaymentFor**m.js: 카드 정보 입력, 실시간 마스킹 로직, 유효성 검사 및 데이터 저장 담당

- **App.css**: 시안 기반의 반응형 레이아웃 및 카드 컴포넌트 디자인 스타일링

## 테스트 방법

- **UI 테스트**: '구매' 버튼 클릭 시 보유 카드 화면으로 정상 이동하는지 확인

- **입력 테스트**: 카드 번호 입력 시 4자리 단위 띄어쓰기 및 9자리부터 마스킹이 적용되는지 확인

- **유효성 테스트**: 모든 필드(번호, 만료일, 영문 이름, CVC, 비밀번호) 입력 시에만 '작성 완료' 버튼이 활성화되는지 확인

## 유의사항 및 알려진 이슈

- **입력 제한**: 카드 소유자 이름은 영문 대/소문자와 공백만 입력 가능하도록 제한되어 있습니다.

- **카드 마스킹**: 입력창에서 실시간으로 마스킹을 처리할 때 리액트 상태 업데이트와 충돌을 방지하기 위해 정규표현식 기반의 필터링 로직을 사용했습니다.
