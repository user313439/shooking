## 📝 고객 요구사항 정리

> 고객이 장바구니에 담은 상품을 안전하고 편리하게 결제할 수 있도록 다음의 요구사항을 준수합니다.

1. **사용자 편의성:** 카드 번호, 만료일 등을 입력할 때 자동으로 다음 칸으로 이동하거나 가이드가 제공되어야 합니다.
2. **보안성:** 카드 번호 중간 자리 및 비밀번호, 보안코드(CVC)는 마스킹 처리(예: ****)하여 외부 노출을 방지합니다.
3. **직관적인 UI:** 사용자가 현재 어떤 카드를 등록하고 있는지 시각적으로 확인할 수 있는 카드 미리보기 기능을 제공합니다.
4. **유효성 검사:** 잘못된 형식의 번호나 만료일이 입력되지 않도록 실시간으로 체크하여 사용자에게 알립니다.

## 📋 기능 목록 및 컴포넌트 설계

### 1. 결제 수단 선택 및 구매 버튼
* **기능:** 최종 결제 금액 확인 및 결제 프로세스 시작
* **컴포넌트:** `<PaymentSubmitButton />`
* **예외 처리:** 필수 약관 미동의 시 결제 진행 불가 안내

### 2. 카드 번호 입력 및 마스킹
* **기능:** 16자리 카드 번호 입력 및 보안을 위한 일부 번호 별표(*) 처리
* **컴포넌트:** `<CardNumberInput />`
* **예외 처리:** 숫자 이외의 문자 입력 방지

### 3. 만료일(MM/YY) 입력
* **기능:** 카드 유효기간(월/연도) 입력 및 자동 슬래시(/) 삽입
* **컴포넌트:** `<ExpirationDateInput />`
* **예외 처리:** 월(MM) 범위(01~12) 초과 시 경고 메시지 출력

### 4. 카드 소유자 이름 입력
* **기능:** 영문 대문자 위주의 소유자 성명 입력 (최대 30자)
* **컴포넌트:** `<CardholderNameInput />`
* **예외 처리:** 글자 수 초과 시 입력 제한 및 안내

### 5. 보안코드(CVC/CVV) 입력
* **기능:** 카드 뒷면 3자리 숫자 입력 및 보안 마스킹
* **컴포넌트:** `<SecurityCodeInput />`
* **예외 처리:** 숫자 3자리 미만 입력 시 결제 시도 차단

### 6. 카드 비밀번호 입력
* **기능:** 비밀번호 앞 2자리 입력 및 마스킹
* **컴포넌트:** `<PasswordInput />`
* **예외 처리:** 비밀번호 불일치 시 재입력 유도
