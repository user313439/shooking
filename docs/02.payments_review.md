# 결제 모듈 리뷰

## 고객사 요구사항

- **카드 번호 입력**: 16자리 숫자 입력 및 4자리 단위 띄어쓰기 표시
- **마스킹 처리**: 보안을 위해 카드 번호의 9~16자리 실시간 마스킹(`*`) 처리
- **만료일 입력**: `MM / YY` 형식의 자동 포맷팅 기능
- **소유자 이름**: 영문 대문자 변환 표시 및 영문/공백 외 입력 제한
- **유효성 검사**: 모든 필수 항목 입력 시에만 하단 버튼 활성화 (배경색 변경)
- **화면 전환**: 
  - 구매 클릭 시 '보유 카드' 목록으로 이동
  - '보유 카드'에서 '+' 클릭 시 '카드 추가'로 이동
  - 좌측 상단 뒤로가기(`←`) 및 우측 상단 닫기(`✕`) 버튼 동작

## 요구사항에 따른 기능 동작 확인

- **카드 번호 실시간 마킹**: 9번째 숫자 입력 시점부터 실시간으로 `*` 처리가 되어 보안을 유지함을 확인했습니다.
- **입력 제한 로직**: 소유자 이름 칸에 한글이나 숫자를 입력하려 할 때 입력되지 않고 영문만 허용됨을 확인했습니다.
- **버튼 상태 변경**: 16자리 번호, 유효한 만료일, 이름, CVC, 비밀번호 2자리가 모두 채워졌을 때만 '작성 완료' 버튼이 노란색/검은색으로 활성화됩니다.
- **보유 카드 상태 관리**: 카드가 0개일 때는 "새로운 카드를 등록해주세요" 문구가 노출되며, 등록 후에는 해당 문구가 사라지고 등록된 카드가 리스트에 추가되는 것을 확인했습니다.
- **네비게이션**: 각 화면의 닫기(`✕`) 버튼 클릭 시 초기 상품 목록 화면으로 정상 복귀합니다.
