# 📖 개발자 가이드라인 (Developer Guideline)

## 1. 프로젝트 개요
> **SHOOKING MALL 커머스 기능 연동 및 고도화**

* **소개**: 본 프로젝트는 기존의 단일 페이지 쇼핑몰 UI를 확장하여 실제 이커머스 서비스와 유사한 흐름(목록→상세→장바구니→결제)을 구현한 리액트 기반 웹 애플리케이션입니다.
* **목표**: 사용자 경험(UX) 향상을 위한 자연스러운 페이지 전환과 브라우저 새로고침 시에도 구매 데이터가 유실되지 않는 견고한 데이터 흐름을 구축하는 것을 목표로 합니다.
* **해결한 문제**:
    * `useState` 기반의 조건부 렌더링에서 발생하는 URL 공유 불가 및 뒤로가기 오작동 문제 해결.
    * 상태 관리 로직을 최상단(`App.js`)으로 끌어올려(Lifting State Up) 페이지 간 데이터 동기화 구현.

---

## 2. 시스템 아키텍처
> **리액트 라우터 기반 컴포넌트 구조**

시스템은 주소(URL)를 기준으로 각 독립된 컴포넌트가 렌더링되는 구조입니다. 모든 데이터는 `App.js`에서 공급받는 **단방향 데이터 흐름**을 따릅니다.

* **라우팅 구조**:
    * `/` : `ProductList` (상품 목록)
    * `/detail/:id` : `ProductDetail` (상세 정보 및 관련 상품 추천)
    * `/cart` : `Cart` (장바구니 및 배송비 계산)
    * `/wallet` : `Wallet` (카드 선택 및 결제 승인)
    * `/success` : `Success` (결제 결과 리포트)

---

## 3. 개발 환경 설정 및 규칙
### 🛠 환경 설정
1. **Node.js**: v16 이상 권장
2. **패키지 설치**: `npm install` (특히 `react-router-dom` 설치 필수)
3. **실행**: `npm start`

### 📋 개발 규칙
* **컴포넌트 명명**: `PascalCase` 사용 및 파일명과 컴포넌트명을 일치시킴 (예: `Cart.js`).
* **브랜치 전략**: `main` 브랜치를 기준으로 하며, 기능별 작업 시 `feature/기능명` 브랜치 활용 권장.
* **커밋 메시지 규칙**:
    * `feat:` 새로운 기능 추가
    * `fix:` 버그 수정
    * `docs:` 문서 수정
    * `style:` 코드 의미에 영향을 주지 않는 수정 (CSS 등)

---

## 4. 프로젝트에서 중요했던 지점
* **라우팅 아키텍처 도입**: 단순 상태 변경 방식에서 `react-router-dom`으로 전환하여 서비스의 확장성을 확보했습니다.
* **동적 라우팅과 파라미터**: `useParams`를 활용해 하나의 상세 페이지 컴포넌트로 여러 상품 데이터를 처리할 수 있는 구조를 잡았습니다.
* **데이터 정합성**: 결제 금액 데이터가 결제 완료 페이지(`Success`)까지 정확히 전달되도록 `location.state`를 활용했습니다.
